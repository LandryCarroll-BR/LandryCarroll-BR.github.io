Creating Digital Signage
From the Ground Up

How to develop quick solutions that can
adapt over time.

Overview

A Local religious organization is seeking to redesign and restructure the information screens displayed throughout their building.

Problem

Since the client was using the same CMS that manages their website to update their information screens, they couldn’t make real-time updates. So to see content, each screen had to be refreshed to display the latest information.

The client used an image slider to display content, requiring a designer to create a separate image for every new announcement or piece of content added. Although this is not too complex a system to maintain, they did not have the proper staff to create and maintain the images required to meet the demand of incoming updates.

Desired Outcome

The client wanted someone to quickly and easily update the content on their information screens without needing any design skills or knowledge. They also required an update to their system as soon as possible, as they had a sizeable upcoming event and wanted to have the new system ready for then.

Solution

Based on my experience using React-Redux to develop web apps and Google Firebase’s Firestore database to make real-time updates, implementing these technologies would make an excellent solution for a quick expedition of the final product.

Future Development

The client is open to working together again on this project in the future to add more features and quality-of-life improvements. These possibilities include using the Pexels API for finding images within the application and an edit function.
